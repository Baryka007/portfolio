C# Server‚ÄìClient Asynchronous Handler App

This project is a system built from a C# WPF client and a console-based server, communicating via the TCP protocol. The client and server send commands back and forth, separated by a ‚Äò;‚Äô delimiter.

On initial connection, the server generates a unique port for each client, allowing it to handle thousands of simultaneous connections. It's a clean and efficient approach to scalable communication.

üéØ Not a complex architecture ‚Äî and that‚Äôs exactly the point.
The system is simple, stable, and maintainable, and does exactly what it‚Äôs meant to: automatically deliver games, files, or updates to client machines.

üí° I learned a lot through this project:

  • about asynchronous TCP communication
  • about port management and resource control
  • about building user-friendly client UIs (in WPF)
  • and that sometimes simplicity is the most professional solution

The following example shows how the server starts up: after loading the necessary data (like a client list), it enters a while loop to continuously listen for incoming connections.

When a client connects, the server asynchronously launches a new function dedicated solely to handling that specific client. This allows the main loop to keep accepting new connections without interruption, while each client gets its own dedicated "worker thread".
